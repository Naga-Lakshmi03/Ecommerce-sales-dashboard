# E-commerce Sales Analysis (Interactive Dashboard Creation Using Power BI)
## Project Overview 
Developed an interactive US E-Commerce Sales Dashboard in Power BI to analyse Year-to-Date sales, profit, product performance, regional trends, and shipping preferences. The dashboard provides actionable business insights through KPI tracking, YoY comparisons, and dynamic visualizations to support data-driven decision-making.

## Dataset Used
- <a href="https://github.com/Naga-Lakshmi03/Ecommerce-sales-dashboard/blob/main/ecommerce_data.csv">Data Set 1</a>
- <a href="https://github.com/Naga-Lakshmi03/Ecommerce-sales-dashboard/blob/main/us_state_long_lat_codes.csv">Data Set 2</a>
## Problem Statement 
An e-commerce company wants to understand its sales performance, customer purchasing behaviour, and profitability across different regions, categories, and time periods. Management needs insights to identify high-performing areas, improve weak-performing segments, and make data-driven business decisions.

## Key Performance Indicator’s
- Create a KPI Banner showing YTD Sales, YTD Profit, YTD Quantity sold, YTD Profit Margin.

- Find Year on Year growth for each KPI and show a YTD sparkline for each measure in the KPI to understand the monthly trend for each fact.

- Find YTD Sales, PYTD Sales, YOY Sales growth for different customer category. Add a trend icon for each category.

- Find YTD Sales Performance by each state.

- Top 5 and Bottom 5 Products by sales.

- YTD Sales by Region to know best and worst performing region all over country.

- YTD Sales by Shipping Type to get the best Shipping type percentage.

## Process 
## Understand the Business Problem
Identified the business objective to analyse sales performance, profit, customer behaviour, and regional trends to support better business decisions. 
-	Collect the Dataset
Gathered the e-commerce sales dataset containing details such as orders, customers, products, regions, sales, profit, and discounts. 
-	Import the Data
Loaded the dataset into Power BI (or Excel) for analysis and dashboard creation. 
-	 Clean the Data
Removed duplicate records, handled missing values, corrected data types, and fixed inconsistent entries to ensure accurate analysis. 
-	 Transform the Data
Created additional columns such as Year, Month, Quarter, and Profit Margin to make analysis easier and more meaningful. 
-	Perform Data Analysis
Explored the data to identify sales trends, customer purchasing patterns, product performance, and regional performance. 
-	 Create Key Performance Indicators (KPIs)
Calculated important metrics like Total Sales, Total Profit, Total Orders, Average Order Value, and Profit Margin to measure business performance. 
-	Design Visualizations
Built charts and graphs to display sales by region, category, state, monthly trends, top products, and profit distribution. 
-	Add Interactive Filters
Included slicers for Year, Month, Region, Category, and State so users can interact with the dashboard and analyse specific data. 
-	Validate the Dashboard
Cross-checked dashboard values with the original dataset to ensure all calculations and visualizations were accurate. 
-	Identify Business Insights
Analysed the dashboard to discover top-performing regions, profitable product categories, seasonal sales trends, and areas needing improvement. 
-	Provide Recommendations
Suggested actions such as increasing focus on profitable categories, optimizing discounts, improving low-performing regions, and strengthening customer retention strategies. 
-	Prepare the Final Dashboard
Organized the dashboard with a clean layout, clear KPIs, charts, and filters to make it easy for stakeholders to understand. 
-	Review the Results
Verified that the dashboard answered the business questions and provided meaningful insights for decision-making. 
-	Present the Findings
Shared the dashboard and summarized the key insights and recommendations to help management make data-driven business decisions
